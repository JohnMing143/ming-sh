# ming.sh

オールインワンの Linux サーバー管理ツールボックスです。インストール後は
`m` コマンドひとつで、対話式メニューからシステムメンテナンス、Docker、
Web サイト構築、ネットワーク最適化などの日常的な運用作業を行えます。

[简体中文](README.md) · [繁體中文](README.tw.md) · [한국어](README.kr.md)

> [!WARNING]
> 本ツールには、ファイアウォール、ネットワーク、SSH、cron、Docker、systemd、
> パッケージ、および `/etc`、`/usr`、`/home` 配下のファイルを変更する高権限
> 機能が含まれます。まずコードを確認し、テスト機で検証したうえで、適切な権限で
> 実行してください。リモートスクリプトを直接 shell に渡さないでください。

## 主な機能

- **システム管理**: システム情報の表示、システム更新、不要ファイルの掃除、基本ツール
- **Docker 管理**: コンテナ・イメージ・ネットワーク・ボリュームの導入と保守、定番アプリのワンクリック展開
- **Web サイト構築・運用**: LNMP 環境、サイト管理、SSL 証明書、バックアップと移行
- **ネットワーク最適化**: BBR 高速化、カーネルチューニング、ファイアウォールとポート管理
- **実用ツール**: SSH 防御、ディスク管理、rsync 同期、クラスター管理、バックグラウンドタスク
- **追加モジュール**: OpenClaw、ゲームサーバー（Palworld、Minecraft）などの補助機能

## インストール

スクリプトを先にローカルへダウンロードし、内容を確認してから実行することを
推奨します（`curl ... | bash` は使わないでください）:

```bash
curl -fL --output ming.sh \
  https://raw.githubusercontent.com/JohnMing143/ming-sh/main/jp/ming.sh
bash -n ming.sh
less ming.sh
bash ming.sh
```

初回実行時にコマンドが `/usr/local/bin/m` にインストールされ、以後は任意の
ディレクトリで `m` と入力するだけでメインメニューが開きます。

他の言語版を使う場合は、ダウンロードパスの `jp/ming.sh` を次のいずれかに
置き換えてください:

```text
ming.sh       メイン実装（简体中文）
cn/ming.sh    简体中文
en/ming.sh    English
kr/ming.sh    한국어
tw/ming.sh    繁體中文
```

## 使い方

```bash
m
```

メニュー番号を選ぶだけで各機能を利用できます。各機能は実行前に行う操作を
説明します。依存関係のダウンロードやシステム変更を伴う操作は、対応する
メニュー項目を自分で選択したときにのみ行われます。

システムチューニング関連の機能は `/etc/sysctl.d/99-ming-sh-*.conf` 設定
ファイルと `# ming-sh-optimize` マーカーを使用するため、識別と後始末が
容易です。

## プライバシーとセキュリティのデフォルト

- **利用統計なし**: プロジェクトのソースコードにはテレメトリや統計送信の
  ロジックは一切含まれません。
- **自動更新なし**: プロジェクトの自己更新と自動更新 cron はいずれも無効で、
  更新は完全にユーザーの管理下にあります。
- **GitHub へ直接接続**: デフォルトでは GitHub プロキシを使用しません。
- **依存元の透明性**: 一部の Nginx テンプレート、Docker Compose ファイル、
  Docker イメージなどは上流プロジェクト由来です。これらのアドレスはすべて
  [`config/project.conf`](config/project.conf) の `UPSTREAM_*` 変数に集約されて
  おり、完全な一覧は [`SECURITY_AUDIT.md`](SECURITY_AUDIT.md) を参照してください。
- **開発用翻訳スクリプトはデフォルト無効**: このスクリプトはソースコードの
  断片を Google Translate に送信するため、明示的に
  `ALLOW_REMOTE_TRANSLATION=true` を設定した場合にのみ実行されます。通常の
  利用では関係ありません。
